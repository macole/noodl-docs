# ライブラリの概要

Noodlには箱から出してすぐに利用可能な多くのノードがありますが、すべてのものに対してノードを含めることはできません。そのため、特定の機能やUIコントロールをキャプチャするノードやコンポーネントを含むモジュールやプリファブでプロジェクトを拡張できます。

**モジュール**と**プリファブ**の主な違いは、モジュールがエディターに新しいノードをインストールするのに対し、プリファブはコアノードで構築されたコンポーネントをクローンし、自分で作成したかのように保存することです。プリファブは何度でもインポートでき、各クローンは好きなように変更できます。

## プリファブ

Noodlのプリファブライブラリは、データリッチなアプリケーションを迅速に構築するために最適化されています。そのまま使用するか、内部の構築ブロックをカスタマイズして独自のものにします。

## モジュール

モジュールは、最も一般的にはプロジェクトに新しい**コアノード**を追加することによって、プロジェクトを何らかの形で拡張します。これらのノードはノードピッカーに追加され、Noodlの他のコアノードと同様に機能します。
